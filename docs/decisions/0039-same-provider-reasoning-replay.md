# ADR-0039: Same-provider signed-reasoning replay — a behavioral amendment to ADR-0030

- **Status**: Accepted
- **Date**: 2026-06-14
- **Related**: [ADR-0030](0030-llm-seam-shape-amendment-reasoning-response-format-provider-executed.md) (amends), [ADR-0011](0011-internal-llm-abstraction.md), [ADR-0038](0038-agentrunner-llm-call-boundary.md), [llm-provider-seam.md](../reference/shared-core/llm-provider-seam.md), [code-style-typescript.md](../standards/code-style-typescript.md), [testing.md](../standards/testing.md), [deferred-tasks.md](../roadmap/deferred-tasks.md)

## Context

[ADR-0030](0030-llm-seam-shape-amendment-reasoning-response-format-provider-executed.md) amended the `@relavium/llm` seam **shape** to carry a reasoning channel: a canonical `{ type: 'reasoning', text, signature?, redacted? }` `ContentPart` ([content.ts](../../packages/shared/src/content.ts):447-450) and the `reasoning_start`/`reasoning_delta`/`reasoning_end` `StreamChunk` triad, where `signature` is the provider's **ephemeral same-provider continuity token**. It was explicitly shape-only, and ADR-0030 already stated the *intent* that "only the originating adapter feeds it back." This ADR supplies the missing **replay behavior** and corrects three things a careful read of the merged code shows are not yet true.

Workstream **1.O**'s headline acceptance includes an agent tool-call **round-trip**. On a thinking-capable provider this is an *interleaved-thinking continuation*: the assistant emits a (signed) thinking block, then a `tool_use`; the engine runs the tool and issues the **next** request carrying the prior assistant turn. **Anthropic rejects (HTTP 400) a continuation whose prior signed thinking block was dropped.** Verified against the code, the path is broken on three axes:

1. **The Anthropic adapter drops every reasoning part on the way out, at the type level.** `toAnthropicBlock`'s parameter type is `Exclude<ContentPart, { type: 'reasoning' } | { type: 'media' }>` and `toAnthropicMessage` filters reasoning (and media) parts out **before** the `.map(toAnthropicBlock)` ([anthropic.ts](../../packages/llm/src/adapters/anthropic.ts):209,212-213,246-250). It is *type-level impossible* to replay a signed thinking block today, and widening only `toAnthropicBlock` would change nothing — the `.filter()` removes the part first.
2. **The strip-latch does not survive the tool loop — so the hole is *multi-turn*, not single-call.** The tool loop is a sequence of separate `generate()`/`stream()` calls (run the tool, re-issue with the `tool_result`). `#lastProvider` — the gate that strips reasoning on a provider boundary — is a **`ChainRun`** field (:581), and a fresh `ChainRun` is created on **every** call (:249,:308), so it resets each call. `#cooldownUntil` is a chain-instance field (:212) but only `rate_limit` sets a cooldown (:453); a retryable timeout/transport/overloaded failover sets **none**. So: continuation #1 fails over from primary **P** to **Q**, which emits **Q-signed** reasoning + a `tool_use`; 1.O re-issues; continuation #2 starts a fresh `ChainRun` at **P** again (P not cooled down, `#lastProvider = undefined`) → **no strip** → **Q's signed reasoning is sent to P** — exactly the cross-provider replay ADR-0030 means to prevent. A single-`generate()` test never exercises this.
3. **"Surviving reasoning part ⇒ replay-safe" is not provider-general, and the canonical content does not faithfully carry every provider's opaque continuation token.** The Anthropic **signed** thinking case works with the existing `signature` field, but: (a) `redacted_thinking`'s opaque `data` is **dropped** on the way in (`{ type: 'reasoning', text: '', redacted: true }` — [anthropic.ts](../../packages/llm/src/adapters/anthropic.ts):126-127), so a redacted block can never be reconstructed; and (b) **Gemini** carries `thoughtSignature` on **any** `Part` including a `functionCall`, and the adapter drops it (`mapContent` reads only name/args — [gemini.ts](../../packages/llm/src/adapters/gemini.ts):193-198) while the canonical `tool_call` part has no field for it ([content.ts](../../packages/shared/src/content.ts):419-441). Gemini 3 function-calling continuations can themselves 400 without the returned signature. So replay correctness is **per-provider** and depends on the canonical content faithfully preserving that provider's opaque metadata on the right parts and in order.

Reasoning is **not** a `FallbackChain` skip capability: `requiredCapabilities` maps only `tools` (and, at 1.AF, media) — never reasoning ([capabilities.ts](../../packages/llm/src/capabilities.ts):24-36) — and it **should not**, because a provider that cannot accept a foreign continuation can still be used by *stripping* the reasoning. Continuation compatibility is a **provider-transition** concern (strip on a boundary), not a reason to skip a provider.

This is a non-trivial cross-package behavioral change to an Accepted ADR's seam, so — ADRs being append-only — it is recorded as its own amending ADR.

## Decision

**We will amend ADR-0030's reasoning channel with a replay *behavior* scoped to the case that satisfies 1.O's acceptance — Anthropic signed (non-redacted) thinking — built on three corrections, with the harder per-provider cases explicitly deferred and recorded, not silently shipped half-built.** The invariant is restated precisely: *a reasoning part replays safely to a provider call only when (a) it survives the chain's cross-provider strip — i.e. it is that provider's — and (b) the canonical content faithfully carries that provider's opaque continuation token. Where (b) does not yet hold, that provider's replay is out of scope and recorded as a follow-up.*

Three corrections, all landing with 1.O ([ADR-0038](0038-agentrunner-llm-call-boundary.md)) in the **same PR or behind a hard acceptance gate**:

- **Make the strip-latch persist across the tool loop (1.K change-set, no new ADR).** Move `#lastProvider` from the per-call `ChainRun` onto the **`FallbackChain` instance**, set to the provider whose result is currently carried in the request (the last *attempted, non-skipped* entry). On the next call's first attempted entry, strip the reasoning when that entry's provider differs from the persisted `#lastProvider`. With [ADR-0038](0038-agentrunner-llm-call-boundary.md)'s one-chain-per-node-execution, this makes the strip correct across continuations: continuation #2 starting at **P** with persisted `#lastProvider = Q` strips Q's reasoning before P sees it. (Residual edge: if the carried provider's reasoning is stripped at an earlier failed entry and the chain then settles back on that provider, same-provider replay is *lost* but never *wrong* — degraded, not a 400; recorded as an acceptable edge.) The mandatory regression spans **two consecutive chain calls** — a single-call test cannot see this.
- **Lower a surviving reasoning part in the Anthropic adapter — at both sites.** Widen `toAnthropicBlock`'s parameter type to admit `{ type: 'reasoning' }`, **and** stop filtering reasoning out in `toAnthropicMessage` (:246-250), **and** update the now-stale comment at :209. Reconstruct the wire block from a surviving **signed** part as `{ type: 'thinking', thinking: text, signature }`. The adapter does **not** test provider identity (it is stateless and provenance-blind — it sees one `LlmRequest`); it trusts the chain to have stripped anything cross-provider.
- **Carry the reasoning part through, in the engine.** The AgentRunner builds the assistant turn from the `LlmResult` *including* its signed reasoning `ContentPart` (on the streaming path, folding `reasoning_start`/`reasoning_delta`/`reasoning_end` into that part, the `signature` arriving on `reasoning_end`, mirroring the non-streaming `mapContent`) and re-feeds it on the same-provider continuation ([ADR-0038](0038-agentrunner-llm-call-boundary.md) names this the turn-core contract). The chain strips it on a cross-provider failover.

**Explicitly deferred and recorded ([deferred-tasks.md](../roadmap/deferred-tasks.md)), not silently shipped:**

- **Anthropic `redacted_thinking` replay** — faithful replay needs the opaque `data` the inbound fold currently drops; carrying it requires extending the canonical reasoning `ContentPart` with an opaque continuation field (a seam-shape addition, an ADR-0030 follow-up). Until then a `redacted` part is carried as today (`text: ''`, `redacted: true`) and **not** lowered back; redacted-thinking continuations are out of 1.O scope.
- **Gemini part-level `thoughtSignature`** (including on `functionCall`) — needs a continuation-metadata carrier on the canonical `tool_call` (and reasoning) parts and adapter capture/replay; deferred to the Gemini reasoning-replay follow-up. Until then Gemini reasoning/tool continuations do not replay the signature.

**Acceptance / regression (recorded-fixture, per the adapter conformance discipline):** (1) an Anthropic interleaved-thinking **+ tools** continuation replays the signed block on the same provider and succeeds; (2) a **multi-turn** sequence where continuation #1 fails over P→Q (a *non-cooldown* retryable error), Q emits signed reasoning + a tool call, the tool runs, and continuation #2 **strips** Q's reasoning before the next attempt on P and stays well-formed (the BLOCKER-1 regression — two consecutive chain calls). The cross-provider strip must be tested alongside the same-provider replay.

Considered alternatives:

- **Reuse the chain instance and assume `#lastProvider` survives** (rejected) — it is a `ChainRun` field reset every call; reuse fixes cooldown, not the latch. The latch must move to instance scope.
- **The adapter lowers reasoning only when "the request stays on the originating provider"** (rejected) — the adapter is stateless and provenance-blind; the same-vs-cross knowledge lives in the chain (`#lastProvider`). The adapter unconditionally lowers whatever survives.
- **Treat reasoning as a `FallbackChain` skip capability** (rejected) — `requiredCapabilities` does not (and should not) gate on reasoning; a provider that cannot take a foreign continuation is still usable by stripping. Skipping it would wrongly eliminate a healthy fallback.
- **Add the full opaque-continuation carrier (redacted `data` + Gemini part signatures) now** (rejected for 1.O) — a broader seam-shape change across `@relavium/shared` content + all adapters; out of proportion to 1.O's headline (the common signed-thinking+tools case). Recorded as follow-ups rather than built half-way.
- **Descope the thinking+tools continuation from 1.O** (rejected) — the tool round-trip with reasoning is 1.O's headline acceptance; shipping it without same-provider replay leaves a silently-broken interleaved-thinking path on the flagship provider.

## Consequences

### Positive

- The Anthropic interleaved-thinking **+ tools** continuation works on the originating provider **across a multi-turn tool loop and after a failover**, closing both the lowering gap and the strip-latch-lifetime hole ADR-0030 left.
- The adapter becomes **stateless-correct** (lowers whatever survives); **one place — the chain instance — owns the same-vs-cross decision**, and one place — the AgentRunner's single per-node-execution chain — gives it a stable lifetime.
- No reasoning **skip** regression: a healthy fallback provider is never eliminated for a reasoning capability it does not need.

### Negative

- **Cross-package coupling**: 1.O (`@relavium/core`) depends for its headline acceptance on two `@relavium/llm` changes — the Anthropic adapter lowering **and** the 1.K `#lastProvider` instance-scope move. All land in the **same PR or behind a hard acceptance gate**, or 1.O stays green against stub providers while a real Anthropic run 400s.
- The replay is **provider-specific, not general**: redacted-thinking and Gemini part-level signatures are **explicitly out of scope** and recorded as follow-ups; until they land, those continuations do not replay their opaque tokens. The "no seam-shape change" property holds for the in-scope **signed Anthropic** case only — the deferred cases will each need a shape addition (tracked against ADR-0030).
- Moving `#lastProvider` to instance scope is a small but real 1.K behavior change with its own two-call regression and a documented residual edge (strip-then-same-provider-retry loses replay but never sends a foreign signature).
