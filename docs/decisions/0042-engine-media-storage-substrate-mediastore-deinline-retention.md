# ADR-0042: Engine media storage substrate — the `MediaStore` host port, the `deInlineMedia` choke-point ordering, and the `media_objects` retention/GC store (amends ADR-0036)

- **Status**: Accepted
- **Date**: 2026-06-18
- **Related**: [0036-run-loop-substrate-event-bus-and-execution-host.md](0036-run-loop-substrate-event-bus-and-execution-host.md) (**this ADR amends it** — append-only, ADR-0036 is unchanged in history; it adds the `MediaStore` slot to the `ExecutionHost` seam and pins the async-`deInlineMedia` ordering at the single producer-side translation point), [0031-llm-seam-shape-amendment-multimodal-io.md](0031-llm-seam-shape-amendment-multimodal-io.md) (the multimodal seam amendment whose reserved `MediaStore`/`deInlineMedia` shape + handle-only `DurableMediaPart` + I3 this *wires*; the retention/GC default this *promotes from a tracked default to a decision*), [0003-pure-ts-engine-not-langgraph-python.md](0003-pure-ts-engine-not-langgraph-python.md) (the derived-from-`run_events` checkpoint rule this stays compatible with — `media_objects` is the **first persisted mutable state outside** that model), [0005-sqlite-drizzle-local-postgres-cloud.md](0005-sqlite-drizzle-local-postgres-cloud.md) (one Drizzle schema, SQLite↔Postgres parity), [0032-desktop-rust-media-de-inline-amends-0018.md](0032-desktop-rust-media-de-inline-amends-0018.md) (the desktop Rust CAS this contract is the platform-free twin of), [0043-media-egress-failover-rematerialization-ssrf.md](0043-media-egress-failover-rematerialization-ssrf.md) (the egress/failover sibling that consumes the same `MediaStore`), [0044-media-access-governance-read-media-save-to-cost.md](0044-media-access-governance-read-media-save-to-cost.md) (the access/cost sibling that reads `media_objects`), [../reference/desktop/database-schema.md](../reference/desktop/database-schema.md) (the `media_objects` canonical DDL home), [../reference/contracts/sse-event-schema.md](../reference/contracts/sse-event-schema.md), [../analysis/multimodal-io-design-2026-06-07.md](../analysis/multimodal-io-design-2026-06-07.md) (decisions B1, the retention default).

## Context

Workstream **1.AF** (Engine media plumbing — Phase C of the 1.m6 multimodal sub-spine) wires *behavior* onto the seam *shape* that [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) froze at 1.AD and the media-input adapters that landed at 1.AE. Three of the seam shapes were landed **reserved, implementation-deferred to 1.AF**: the `MediaStore` contract (`put`/`get`/`resolveForEgress`, `Uint8Array`-shaped, [`content.ts`](../../packages/shared/src/content.ts) ~631–664), the `deInlineMedia` overloads (the in-flight→durable transform), and the `media_objects` retention/GC table. ADR-0031 recorded the latter two only as a **default in its "Open implementation details"** ("per-distinct-reference refcount + `last_referenced_at` + a grace window … the `media_objects` table lands in Phase C (1.AF)") — a default, not a decision. 1.AF must now make three genuinely-undecided choices that no ADR covers, each of which a wrong guess forces a re-plumb every surface re-implements:

1. **Where the `MediaStore` host port is injected, and the async-`deInlineMedia` ordering at the choke point.** [ADR-0036](0036-run-loop-substrate-event-bus-and-execution-host.md) pinned the `ExecutionHost` seam *in principle and its Phase-1 slices only* (clock/timer, the persistence/store port, the transport boundary), explicitly leaving "the exhaustive method surface … [for] its `docs/reference/` home." It has **no media slot**. ADR-0036 also fixed the run loop's single producer-side translation point: exactly one place masks secrets, assigns the single authoritative `sequenceNumber`, and performs the **synchronous durable write before any subscriber is notified** (persist-before-deliver). Verified in [`engine.ts`](../../packages/core/src/engine/engine.ts): `#emitDurable` is that wrapper, `#bus.next(draft)` both **assigns the `sequenceNumber` and Zod-validates** against `RunOrSessionEventSchema`, and delivery is serialized via `#deliveryTail`. `deInlineMedia` is necessarily **async** (it `put`s bytes into the store, and — per [ADR-0043](0043-media-egress-failover-rematerialization-ssrf.md) — may fetch a provider output URL). Threading an async transform through that synchronous, gap-free, persist-before-deliver chokepoint is a material change to the ADR-0036 contract; a naive `await` could stall the producer stream, reorder events, or break monotonic numbering. (ADR-0031's guardrail says `deInlineMedia` runs at "one serialize/emit choke point" but does not pin *where in the ordering* the async pass sits, nor name the streamed `agent:token` hot path it must not block.)

2. **The `media_objects` retention/GC model.** This is the **first persisted, mutable state outside the [ADR-0003](0003-pure-ts-engine-not-langgraph-python.md) derived-from-`run_events` model** — every other durable concern (checkpoints included) is *derived*; `media_objects` is a real, mutated table. The DB has exactly 11 tables today and **no media table** ([`schema.ts`](../../packages/db/src/schema.ts); [database-schema.md](../reference/desktop/database-schema.md) has no media section). The refcount *grain* (a denormalized counter vs a reference-junction), the grace-window basis, and how the model persists the [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) `read_media` scope-set are undecided.

3. **The terminal-state media sweep.** 1.AF's acceptance demands that "a run reaching a terminal event (`run:completed | failed | cancelled`) deterministically reclaims its media refs … a terminal-state sweep, **not** refcount-GC alone," so a FAILED/CANCELLED run leaves no orphaned partial media. How a platform-free engine signals that sweep, how it composes with the refcount table without double-free or deleting bytes another run still references, and where the boundary sits against the **1.AG** async-job cancel-sweep, are unspecified.

The stakes are high because all three sit on the seam/purity/durability boundaries every surface (CLI filesystem store, desktop Rust CAS, Phase-2 managed store) re-implements. Getting the injection point or the ordering wrong is an ADR-0036-level rewrite; getting the retention model wrong orphans or over-deletes user media.

## Decision

**We record the engine media *storage* substrate for 1.AF as the following pinned choices, as an amendment to (not a supersession of) [ADR-0036](0036-run-loop-substrate-event-bus-and-execution-host.md)** — ADR-0036's run-loop substrate is unchanged; this adds the media slot to the `ExecutionHost` seam and pins the async transform's place in the chokepoint it defined. The egress/failover half is [ADR-0043](0043-media-egress-failover-rematerialization-ssrf.md); the access/cost half is [ADR-0044](0044-media-access-governance-read-media-save-to-cost.md). Canonical DDL lives in [database-schema.md](../reference/desktop/database-schema.md).

### 1. The `MediaStore` is an optional host port on the `ExecutionHost` seam

We add an **optional `readonly mediaStore?: MediaStore`** to `ExecutionHost`, mirroring its existing host concerns (clock/ids/store/checkpointer/`setTimer`). It is **absent-tolerant**: a text-only host (or a run that produces no media) leaves it `undefined` and never enters a byte path. The engine references media **only by the handle string**; bytes are carried as `Uint8Array` (an ES-lib built-in — it keeps `tsconfig.purity.json` `types: []` clean, no `node:Buffer`). The store is **host-injected, never imported** — the same purity seam as the ADR-0036 persistence port and the ADR-0018 HTTP transport. The Phase-1 hosts inject a **filesystem content-addressed store (CAS)** in `@relavium/db` (CLI / VS Code) and an **in-memory reference** for tests; the desktop Rust CAS ([ADR-0032](0032-desktop-rust-media-de-inline-amends-0018.md)) and the managed store are 1.AH.

*Considered:* **(a)** a new capability on the `ToolHost` bundle ([ADR-0037](0037-engine-tool-execution-boundary.md)) — *rejected*: the store is reached at the **emit/persist choke point and on failover**, both run-substrate concerns, not tool-execution concerns; `ToolHost` is scoped to a tool dispatch. **(b)** a third sibling host seam — *rejected*: ADR-0036 already designates `ExecutionHost` as the run-substrate boundary "at which the whole engine relocates" for Phase-2 cloud; the media store relocates with it. **(c, chosen)** `ExecutionHost.mediaStore?` — relocates correctly, mirrors the existing optional ports, stays absent-tolerant.

### 2. `deInlineMedia` runs on the draft *before* the synchronous seq-assign / validate / persist / deliver critical section

The async `deInlineMedia` pass is **hoisted before** the synchronous, serialized critical section ADR-0036 defined; it does **not** split or reorder it:

- `deInlineMedia` runs at `#emitDurable` as the **sole** emit-time transform on the draft (the only new `await`). **Secret masking is not at this choke point** — it already happened upstream (input masking at run setup → `#maskedInputs`, verified `engine.ts:268`; per-node output masking via `secretInputNames` at the node executor, `engine.ts:846`), so the draft reaching `#emitDurable` is already masked. De-inline relates to masking only by *sequence* (masking upstream, then de-inline here), never by sharing a pass — the earlier "composes with the existing `maskInputs` pass at the choke point" framing was incorrect (`#emitDurable`, `engine.ts:1346`, contains no masking call).
- **The binding invariant:** the event that is **validated against the durable schema, persisted to `run_events`, and delivered is handle-only** — so de-inline must run *before* that validation, because the durable schema's typed media positions (e.g. `tool_result.media : DurableMediaPart[]`) are handle-only and would *reject* in-flight base64. The single authoritative `sequenceNumber` stays **gap-free + monotonic**, and **persist-before-deliver** plus the `#deliveryTail` seq-ordered delivery are preserved. `#bus.next` today both assigns the seq **and** validates (`engine.ts:1365`); the 1.AF engine PR pins the exact placement — either de-inline *before* `#bus.next`, or split the synchronous seq-assignment from a post-de-inline validate — so long as **the validated/persisted/delivered event is the handle-only one** and the seq/delivery invariants hold. (The reading that validates/persists a *pre*-de-inline event is rejected.) A test asserts a gap-free monotone sequence under media-bearing events.
- **A no-media fast path** keeps text-only emits free: `deInlineMedia` first does the cheap `containsInlineMediaBytes`-class scan and returns the input unchanged (no `await` cost beyond the scan) when there is no media — so the overwhelming text-only majority pays a scan, not a store round-trip.
- **The streamed `agent:token` hot path is not blocked.** De-inline applies to **durable** emits at `#emitDurable`. Streaming media never rides `agent:token`; it rides the `media_start`/`media_delta`/`media_end` triad whose `media_delta` carries no base64 and whose `media_end` is a handle-only `DurableMediaPart` de-inlined at the durable boundary. The transient streamed `#nodeEmit`→`#bus.emit` path stays synchronous for tokens; a media-bearing streamed frame is de-inlined before it becomes durable, never on the token flood.

The choke point's **coverage set is unchanged from ADR-0031's guardrail** and re-affirmed: `LlmResult.content`, every node output and event payload (`node:completed.output`, `run:completed.outputs`, `run:failed.partialOutputs`, `run:started.inputs`, the `agent:tool_result` descriptor), the **derived checkpoint/`RunState` snapshot** (protected by transitivity — the snapshot derives from already-de-inlined `run_events`, [ADR-0003](0003-pure-ts-engine-not-langgraph-python.md)), and the **FallbackChain cross-provider context transfer** (B1; the seam-side mechanics are [ADR-0043](0043-media-egress-failover-rematerialization-ssrf.md)). One acknowledged exception is recorded, not hidden: `engine.ts` `reconcile()` persists a synthetic `run:failed` **directly via the store, bypassing `#emitDurable`**; it carries **no node output** (only an error code), so it cannot leak media — a backstop test asserts it stays media-free, keeping the "one boundary" claim honest rather than silently false. (Any future widening of `reconcile()` to carry node output **must** route it through `#emitDurable` or run `deInlineMedia` itself; the backstop test guards this — recorded in [deferred-tasks.md](../roadmap/deferred-tasks.md).)

### 3. `media_objects` is a real, host-owned table with a reference-junction refcount

We add `media_objects` (the 12th Drizzle table) **and** a `media_references` **junction** (the 13th — two tables, not one), not a denormalized counter:

- **`media_objects.handle` is a UNIQUE _constraint_** (not merely an index) so the `media_references` foreign key can target it under Postgres too ([ADR-0005](0005-sqlite-drizzle-local-postgres-cloud.md) parity); the content-addressed `media://sha256-<hex>` handle is itself the integrity hash (no separate checksum), and a row soft-deletes via the established `deleted_at` convention (GC byte-reclamation sets it — no new `reclaimed_at` name).
- `media_references(handle, scope_kind, scope_id)` is the **per-distinct-reference junction** the refcount *derives* from, with a **`UNIQUE (handle, scope_kind, scope_id)`** constraint (a scope references a handle at most once). A bare integer counter is **rejected**: it is racy even under SQLite's single writer, it cannot satisfy the "terminal-state sweep, not refcount-GC alone" acceptance (the sweep must be **queryable by run/session**), and it is exactly where the [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) `read_media` scope-set is persisted. **The keying reconciles refcount and authz in one table** (the one cross-ADR seam with [ADR-0044](0044-media-access-governance-read-media-save-to-cost.md)): `scope_kind` is a **superset** — `run` (and `node`) references exist **only for refcount/sweep**, while `session` (and the reserved `workspace`) are **also** the [ADR-0044](0044-media-access-governance-read-media-save-to-cost.md) authz `Scope` kinds. The refcount derives from **all** rows; the terminal sweep (§4) removes the **`run`** rows; `read_media` authz consults **only** the `session`/`workspace` rows — **a `run` reference never grants read**. So a handle may carry both a `run` reference (lifetime) and a `session` reference (authz), and the terminal sweep removing the run row leaves the handle alive while a session row remains.
- **This is the first persisted mutable state outside the ADR-0003 derived model, and stays cleanly separated from it:** `media_objects` is a **retention store, not a checkpoint store**. ADR-0003's "no checkpoint table" rule is untouched — checkpoints stay derived from `run_events`; `media_objects` is a distinct host-owned concern (like the OS keychain is a distinct secret store), and it diverges deliberately from the 90-day `run_events` prune.
- The **canonical column-level DDL + the migration live in their reference home, [database-schema.md](../reference/desktop/database-schema.md) §Media tables** (with the drift-gated fidelity snapshot) — deliberately **not restated here**, to avoid ADR↔reference drift (rule 8). This ADR records the *decisions* (above), not the column list.

### 4. The terminal-state sweep reclaims a run's *references*, ordered before refcount-GC

A run reaching `run:completed | failed | cancelled` triggers a **deterministic terminal-state sweep** through the host `MediaStore` (a `mediaStore.reclaim(runId)`-class port call at the single terminal emit — the engine signals, the **host owns the GC mechanism**, Rust on desktop / filesystem on CLI). The lifecycle is pinned in **three ordered steps**, so bytes another run still references are never deleted and a transiently-zero handle is not reclaimed mid-window: **(a)** the terminal sweep **removes the run's `run`-kind rows from `media_references`** (scoped to the run, never deleting bytes directly); **(b)** a handle whose reference count reaches **zero** enters a **grace window** — a configurable default (**7 days**, recorded as prose in [database-schema.md](../reference/desktop/database-schema.md) §Media tables today, with a formal `[defaults].media_gc_grace_days` config key **forward-declared as a P4/D11 addition** — it does not exist yet), measured from `last_referenced_at`; **(c)** refcount-GC reclaims the bytes (and sets `deleted_at`) only **after** the grace window elapses. This satisfies "not refcount-GC alone" (a FAILED run's partial media is reclaimed by *reference removal at the terminal event*, even if refcount logic alone would have left it) without over-deleting shared media. The **1.AG boundary** is explicit: the minute-scale async-job cancel-sweep (a `pollMediaJob` cancelled mid-flight) is **out of 1.AF scope** and gets its own ADR; 1.AF's sweep/GC/governor design must not foreclose it, so the sweep is keyed on the terminal *event*, agnostic to job lifetime.

## Consequences

### Positive

- The `MediaStore` injection point and the async-de-inline ordering are decided before code, against the exact ADR-0036 chokepoint the engine already implements — so 1.AF wires media without an ADR-0036-level rewrite, and the Phase-2 cloud host swaps the media store with the rest of the `ExecutionHost`.
- The "no bytes durable" invariant (I3) gains its **active** enforcement: with `deInlineMedia` live at the one choke point, the handle-only `DurableMediaPart` type split stops being a latent guarantee and becomes a running transform, with the streamed/`reconcile()` exceptions named and backstopped rather than silently assumed safe.
- The retention model is a queryable, race-safe reference junction that satisfies the terminal-sweep acceptance and gives `read_media` authz a single persistence home — and it is explicitly fenced off from the ADR-0003 derived-checkpoint model so the durability story stays legible.

### Negative

- `media_objects` introduces the project's first mutated durable table — new GC/retention surface to own and test (mitigated: a reference junction with a deterministic terminal sweep, host-owned mechanism, cross-run shared-handle survival tested, and a clean conceptual fence from the derived-checkpoint model).
- The async `deInlineMedia` adds one `await` to the emit chokepoint for media-bearing events and a cheap scan to every emit (mitigated: a no-media fast path; the serialized numbering/persist section is unchanged; the token hot path is untouched).
- The terminal sweep is a new engine→host signal whose mechanism lives in each host (Rust/filesystem) — per-host wiring the Node surfaces and the desktop CAS must each implement consistently (mitigated: the engine-side contract — reference removal at the single terminal event, ordered before refcount-GC — is pinned here once, and the 1.AG cancel-sweep boundary is fenced so the two never double-free).

### Neutral

- Per ADR-0009's append-only rule, ADR-0036 is unchanged in history; this ADR is the authoritative record for the `ExecutionHost.mediaStore` slot and the async-de-inline ordering. Future run-loop readers should read ADR-0036 **and** this.
