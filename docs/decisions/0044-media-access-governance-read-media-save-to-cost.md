## Context

1.AF wires the media *access* and *spend* boundaries onto the [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) seam. Four decisions sit on security or governance contracts that no ADR has settled and a wrong guess "fails open at one forgotten site" (security-review.md) or breaks the cost path for every consumer:

1. **`read_media` byte delivery is a new untrusted byte path.** [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) (A8) pinned the *authz model* (a generic `handle → allowedScopes: Set<Scope>`, owner-equality rejected, sha256-knowledge is not authorization) and security-review.md §Media byte-delivery pins the *gate* (reject negative/reversed/out-of-bounds `Range` against the known `byteLength`; `realpath`+`commonpath` fail-closed; symlinks OFF; a single byte path; stream + size-bound). But **where the gate lives** — engine-pure policy vs host mechanism — is undecided, and `realpath`/`commonpath`/sockets are platform calls forbidden in `packages/core`. The `MediaStore` interface (`put`/`get`/`resolveForEgress`) has **no `read_media`/`Range` method**, and `ToolDispatchContext` carries **no session-scope identity or `byteLength`** to validate against. Guessing duplicates the gate per host — exactly the failure mode the standard warns about.

2. **`save_to` is a new model-bytes write surface.** [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) (A9) said "handle at the seam, `save_to` at the surface" but left `save_to` "**a surface concern … without a contract**." It is a *write* of model-produced bytes to an interpolated relative path — a new security-sensitive surface distinct from the `read_media` read gate, and the platform-pure engine cannot do the write.

3. **The token-based governor cannot price a media call.** [ADR-0028](0028-workflow-resource-governance.md)'s pre-egress governor evaluates `cumulative + worstCaseNextEstimate(maxTokens)`; a `generateMedia` call or a media-output turn has **no token estimate that captures its cost**. The verified `PreEgressHook` signature carries only `{ model, maxTokens? }` — it has **no way to convey** that the upcoming call requests media output or its unit counts. ADR-0031 (A6) said "per-modality flat estimate (config default) + per-model rate," but the **rate shape, the config-default shape, the hook widening, and the realized-cost fold** are unspecified — and the roadmap title ("media budget **caps** + the per-modality pre-egress media cost estimate") is in tension with A6 (cost-estimate-only) over whether a **distinct media count/bytes cap** is in scope.

4. **A binding keychain-bridge acceptance gate has no owner.** 1.AF's acceptance ([phase-1](../roadmap/phases/phase-1-engine-and-llm.md)) includes "the keychain bridge never returns a raw key from an IPC command (a direct test)" — bundled with the media IPC/byte-delivery review surface, but it is a passive invariant with no owning deliverable.

These are governance- and security-contract decisions ([ADR-0028](0028-workflow-resource-governance.md)/[ADR-0029](0029-tool-policy-hardening.md) own the homes; CLAUDE.md rule 9 + the security-review standard apply), so they warrant a recorded decision.

## Decision

**We record the media *access & spend* governance for 1.AF as the following pinned choices, as an amendment to (not a supersession of) [ADR-0028](0028-workflow-resource-governance.md) and [ADR-0029](0029-tool-policy-hardening.md)** — both decisions are unchanged; this adds a media cost class to the governor and the `read_media` authz to the tool policy. The storage substrate is [ADR-0042](0042-engine-media-storage-substrate-mediastore-deinline-retention.md); the egress half is [ADR-0043](0043-media-egress-failover-rematerialization-ssrf.md).

> **Scope note.** This ADR describes the **full** 1.AF access/spend *target state*; 1.AF lands across multiple PRs. The mechanisms here — `read_media` + the byte-delivery gate (D12/D13), the `save_to` write port + the load-time `output_modalities` check (D15/D16), and the per-modality media cost governor (D17) — are the **P4** phase, landing after the P1+P2 substrate (the strict `output_modalities`/`save_to` node fields + OpenAI `mediaUnits` already landed in P1). `read_media`/byte-delivery is security-sensitive and gets its own dedicated security-review pass; the 1.AF acceptance checks (incl. the keychain-bridge no-raw-key IPC test) gate on the PR that lands them.

### 1. `read_media` — one engine-pure scope-set authz primitive, used by both a built-in tool and the surface command

We build **one engine-pure scope-set authz + `Range` primitive** and expose it two ways (per A8): a **model-callable `read_media` built-in** (the 13th `ToolDef`) **and** the 1.AH desktop Tauri `read_media(ref)` command, sharing the one primitive so the gate is never written twice.

- **Authz:** a `Scope` type in `@relavium/shared` (`{ kind: 'session', id }` today; `{ kind: 'workspace', id }` **reserved, documented-not-implemented**). A handle's `allowedScopes` is read from the [ADR-0042](0042-engine-media-storage-substrate-mediastore-deinline-retention.md) `media_references` junction — specifically the **`session`/`workspace`-kind rows only**; the `run`/`node` rows in that junction are refcount/lifetime entries and **never grant read** (the refcount↔authz keying is reconciled in [ADR-0042](0042-engine-media-storage-substrate-mediastore-deinline-retention.md) §3). The membership check runs on the **effective** args. **Owner-id equality is rejected** (it cannot express "received as input"); **knowing a sha256 is not authorization.** The **input-transfer scope-population** is pinned: when a handle is passed as a node/session input, the engine **adds the receiver's session scope** to that handle's `media_references` at the transfer boundary — so a downstream session can read media it legitimately received, and nothing wider.
- **Policy/mechanism split (the core decision):** the **engine owns the policy** — the scope-set membership check and the `Range` math against the **host-populated `byteLength`** carried on a widened tool-dispatch context; the **host owns the mechanism** — `realpath`+`commonpath` fail-closed path resolution, symlinks OFF, and streamed, size-bounded byte I/O (the same delegation as `ResolverCapabilities.readFile`/`EgressCapability.fetch`). The host mechanism is a **new bounded read method on the [ADR-0042](0042-engine-media-storage-substrate-mediastore-deinline-retention.md) `MediaStore` port** (carrying the validated `Range`), reusing the `ExecutionHost.mediaStore?` port — **not** a new `ToolHost` capability arm. **No raw `parseInt` without a bound check**, and **no second raw static mount** beside `read_media`. `read_media` is read-only and **bypasses the Phase-2 `ActionGuard`** ([ADR-0041](0041-external-action-governance-seam.md)).
- **Secret-free denial:** a denial is an **additive** `ToolPolicyDenyReason` member **`media_scope_denied`** (amending [ADR-0029](0029-tool-policy-hardening.md)) — added to the closed union (today **six** members: `not_granted`/`provider_executed`/`command_not_allowed`/`domain_not_allowed`/`insecure_url`/`gate_required`, `tools/errors.ts`). [ADR-0041](0041-external-action-governance-seam.md)'s `governed_block` is a **separate, not-yet-landed** additive member, so there is **no ordering dependency** between them; each exhaustive `switch` consumer must handle every member as it lands (the discriminated-union discipline). The denial adds an `sse-event-schema.md` error-taxonomy entry; the message names a **reason**, never a handle ref, a resolved path, bytes, or a host stack. (The `mimeType` bound — `MediaMimeTypeSchema`, max 255 — matters here: `mimeType` is interpolated into rejection messages, and an unbounded value would turn the rejection *message* into a bytes-smuggling channel into the very logs/events I3 guards.)

*Considered:* a model-callable built-in only (rejected: the desktop display path needs the same gate — it would be written twice); a surface-only command (rejected: a workflow that reads a produced asset needs a model-callable path); building the `Range`/path gate fully host-side (rejected: it duplicates the authz + `Range` math per host — the "forgotten site" failure). **(chosen)** one engine-pure policy primitive + a host mechanism port, used by both.

### 2. `save_to` — a host-injected media-write port; the edge carries the handle

`save_to` is added to `OutputNodeSchema` as a strict ([ADR-0023](0023-strict-authored-yaml-validation.md)) optional **relative** template string (with a new `run.id` interpolation namespace); `output_modalities` is added to `AgentNodeSchema` as a strict optional non-empty array from `OUTPUT_MODALITIES` (`document` and the empty array rejected). The **DAG edge and every run event carry the handle, never bytes**; at the surface boundary the engine resolves `handle → bytes` via `MediaStore.get` and writes through a **host-injected media-write port** with the **same fail-closed path discipline** as `read_media` (`realpath`+`commonpath`, symlinks off, a scope root). The platform-pure engine never does filesystem I/O. 1.AF wires the **port + the handle flow + the load-time membership validation** of `output_modalities` against the model's `media.outputCombinations`. Because `WorkflowSchema.superRefine` has no model catalog, this runs as an **engine-loader pass** — a `validateWorkflowWithCatalog(parsed, catalog)` step using the `@relavium/llm` capability data (a `packages/core`→`packages/llm` *parse-time* dependency, not circular — `core` already depends on the seam) — raising a load-time `WorkflowValidationError`, deferring when the model is unresolvable at parse (never a silent runtime drop). The new `run.id` interpolation namespace + the `save_to` template field are recorded in [workflow-yaml-spec.md](../reference/contracts/workflow-yaml-spec.md) and the interpolation-engine spec (1.L2) as canonical-home updates in the 1.AF PR. The actual surface rendering is 1.AH.

### 3. Per-modality media cost — a disjoint cost class folded into the existing cost cap (no new cap dimension)

We add a media cost class to the [ADR-0028](0028-workflow-resource-governance.md) governor without a new authored cap field. *(The concrete contract shapes below — the `PreEgressHook` fields, the per-model media rate on `ModelPricing`/`model_catalog`, and the `[defaults].media_cost_estimate` shape — are P4/D17; when they land they move to their canonical homes ([config-spec.md](../reference/contracts/config-spec.md), pricing/`model_catalog` in [database-schema.md](../reference/desktop/database-schema.md), and [sse-event-schema.md](../reference/contracts/sse-event-schema.md), which already carries `cost:updated.mediaUnits`). This ADR records the decision + the intended shape until then.)*

- **Widen `PreEgressHook`** from `{ model, maxTokens? }` to also carry `outputModalities?: OutputModality[]` and `mediaUnitsEstimate?: { modality: MediaBilledModality; units: number }[]`; the `AgentRunner`/turn loop populates them from the node's `output_modalities` (unit counts default from `[defaults].media_cost_estimate` when the turn declares no volume). This is the central plumbing change — and a **breaking change to every `PreEgressHook`/`#makePreEgressHook` caller**, all updated in the 1.AF PR.
- The **existing** `MEDIA_BILLED_MODALITIES` (`['image','audio','video']`, `constants.ts`) — a **subset of `MEDIA_MODALITIES`** that excludes `document` (PDF bills as tokens), per [ADR-0031](0031-llm-seam-shape-amendment-multimodal-io.md) decision #4 / A6 — is the closed billed set, and is already what `Usage.mediaUnits.modality` keys on. (No new constant is introduced.)
- **Per-model media rate** on `ModelPricing` (and the `model_catalog` projection): **integer micro-cents** per the table convention — micro-cents **per image**, **per audio-second**, **per video-second** (a single `Math.round` per unit class, no float). **Unit caveat (reconciles 1.AF/D18):** a **token-based** provider reports audio as a raw token **count**, not seconds — OpenAI's `completion_tokens_details.audio_tokens` maps to `Usage.mediaUnits` as `unit: 'count'` with the raw count (NO fabricated tokens→seconds conversion, which would mis-bill). So the realized-cost fold (below) must switch on `MediaUnitsEntry.unit` and price the matching rate; an audio-`count` from such a provider is **observability-only** until a per-count (or a true-seconds) rate exists for that model — never hard-failing a run. A **`[defaults].media_cost_estimate`** config default (authored in [config-spec.md](../reference/contracts/config-spec.md)) is a per-modality **unit-count** default (the analogue of `max_tokens_estimate` — a count, not a price), used when a turn does not declare media-output volume.
- **Estimate:** projected total `= cumulative + token-estimate + media-estimate` (`units × rate`), pre-egress, reusing the **existing** `budget:warning` / `budget:paused` / `BudgetExceededError` / `BudgetPauseError` + `on_exceed` (`fail`/`pause_for_approval`/`warn`) surface — **no new event or error class**. The **unbounded short-circuit** (`max_cost_microcents <= 0 → allow`) stays **before** any estimate (no `/0` in `thresholdPct`); an **unpriced model / missing media rate degrades to allow** (the H4 mirror — never hard-fail a valid run on a missing rate).
- **Realized fold:** `CostTracker.record` folds `Usage.mediaUnits` (image per-count, audio/video per-second) into `cumulativeCostMicrocents` as a **separate addend** (never mixed into the token cost path), so the governor's running total — and the cumulative-cost-on-resume `Math.max` fold — includes realized media spend.
- **No new cap dimension (the contested decision):** the roadmap's "media budget caps" is realized as the **per-modality cost estimate folded into the existing `max_cost_microcents` cap**, **not** a distinct `max_media_count`/`max_media_bytes` field on `BudgetSchema`/`[chat]`. *Considered:* a distinct media count/bytes cap (ADR-0031:231-232's "size+count budget") — *deferred, not adopted*: it is a new authored-contract cap dimension every surface renders, and the cost cap already governs spend; a count/bytes cap can be added **additively** later if a concrete need arises (recorded in deferred-tasks.md). This keeps `BudgetSchema` stable and avoids a speculative contract.

### 4. The keychain-bridge no-raw-key IPC test is an owned 1.AF acceptance gate

The binding acceptance — "the keychain bridge never returns a raw key from an IPC command" — becomes an **explicit test deliverable** in 1.AF (not a passive invariant), bundled with the media IPC/byte-delivery review surface ([ADR-0006](0006-os-keychain-for-api-keys.md) unchanged; this just gives the gate an owner).

### Forward-compat (recorded, not built in 1.AF)

The `MediaStore` contract + `Usage.mediaUnits` + the cost shape must **not foreclose managed mode** ([ADR-0015](0015-managed-mode-data-handling-and-compliance.md)): the gateway is pass-through, stores nothing, meters **counts-not-content**, and generated media materializes to the user's **local** store. Reconciliation is 1.AH; 1.AF's shapes (counts in `mediaUnits`, local `MediaStore`, integer-micro-cent rates) are chosen to be compatible.

## Consequences

### Positive

- The `read_media` byte path lands as **one engine-pure policy primitive + a host mechanism port**, used by both the model-callable tool and the 1.AH desktop command — so the `Range`/`realpath`/`commonpath`/symlink/scope gate is written and tested once, not per host, closing the "forgotten site" failure mode.
- `save_to` becomes a real contract: a handle on the edge, a host-injected fail-closed write port, and load-time `output_modalities` validation — no bytes on the edge, no silent runtime drop, no engine filesystem I/O.
- Media spend becomes a first-class, conservative, pre-egress cost class that **reuses** the ADR-0028 cap/events/`on_exceed` surface (no new event/error/cap to render), degrades-to-allow on a missing rate like the token path, and folds realized `mediaUnits` into the running total — with the cap-dimension question resolved to "no new cap" so `BudgetSchema` stays stable.
- The keychain no-raw-key gate gets an owner, and the managed-mode forward-compat constraint is recorded so 1.AH is not foreclosed.

### Negative

- `read_media` adds an untrusted byte path; a path-traversal/symlink escape, an unvalidated `Range`, or a scope bypass would expose arbitrary bytes (mitigated: `Range` fail-closed against host `byteLength`, `realpath`+`commonpath` + symlinks-off on the host, scope-set membership not owner-equality, a single byte path, a secret-free denial, the `mimeType` bound, and a dedicated security-review pass — shared with the [ADR-0043](0043-media-egress-failover-rematerialization-ssrf.md) egress review).
- Widening `PreEgressHook` and folding `mediaUnits` into `cumulativeCostMicrocents` changes what the cumulative figure *means* for every consumer (`cost:updated`, the checkpoint cumulative restore) — a real cost-path change (mitigated: media is a strictly additive, disjoint addend, never altering the token path; the `Math.max` resume fold is preserved; realized media cost has only OpenAI audio as a live source at 1.AF, so the fold is exercised narrowly and the estimate path carries the weight).
- Deferring a distinct media count/bytes cap means a workflow can only bound media spend via the cost cap in 1.AF (accepted: the cost cap governs spend; a count/bytes cap is additive later and recorded as deferred, avoiding a speculative authored contract now).
- 1.AF widens two seam shapes in one PR: `ToolDispatchContext` gains a **host-populated `byteLength`** and the **session-scope identity** (additive optional fields, like the existing `secretArgKeys?`/`limits?` — existing dispatchers ignore them), and `PreEgressHook` gains `outputModalities?`/`mediaUnitsEstimate?` — so every `ToolHost.dispatch` implementation (CLI / VS Code / desktop / test) and every `#makePreEgressHook` caller is touched (mitigated: the fields are additive/optional and the seams change once, with all callers, in the 1.AF PR).

### Neutral

- Per ADR-0009's append-only rule, ADR-0028 and ADR-0029 are unchanged in history; this ADR is the authoritative record for the media cost class, the `read_media` authz, and the `save_to` write port. Future governance/tool-policy readers should read those **and** this.
